# 오픈뱅킹 검토 기록 (향후 발전 방향용)

> **결론(2026-06 기준)**: 오픈뱅킹이 이 서비스의 "제대로 된" 종착지지만, **개인(비사업자) 신분으로는 운영 불가**하여 현재는 보류한다. 토스 핸드오프 방식으로 컨셉을 검증하고, 사업자/법인·제휴가 확보되는 시점에 오픈뱅킹으로 전환한다. 이 문서는 그때를 위한 사전 조사 기록이다.

## 왜 오픈뱅킹이 답인가

토스 핸드오프 방식의 두 가지 근본 한계를 오픈뱅킹이 모두 해결한다.

| 항목 | 토스 핸드오프 (현재) | 오픈뱅킹 (목표) |
|---|---|---|
| 송금 실행 | 토스 앱이 함(딥링크 전달) | **우리 앱/서버가 직접 이체 API 호출** |
| 받는 사람 계좌 | 매번 말/입력 필요 | **거래내역 조회로 자동 확보** (← 어머니 송금 기록 "불러오기"가 실제로 가능) |
| 송금 완료 확인 | 불가(토스가 안 알려줌) | **API 응답으로 즉시 확인** → 자동 저장·기록 |
| "자주 보내는 사람" | 손으로 등록 | **거래내역 기반 자동 생성** |
| 어르신 부담 | 첫 송금마다 계좌 | 계좌 1회 연결 후 **말로 끝** |

핵심: 처음에 원했던 *"어머니 토스 기록을 불러와 송금 대기로"* 는 토스로는 불가능했지만, **오픈뱅킹 거래내역 조회 API로는 가능**하다.

## 어떻게 동작하나 (개념)

금융결제원이 운영하는 **은행 공동 API**. 사용자가 1회 동의·본인인증으로 계좌를 연결하면, 등록된 이용기관이 표준 API로:
1. **계좌 연결(1회)** — 본인인증으로 위임
2. **거래내역/잔액 조회** — 자주 보낸 사람 자동 채움
3. **이체(출금)** — 서버가 "A계좌→B계좌 N원" 호출 → 즉시 실행 + 결과 수신

## 진입 장벽 (개인이 접는 이유)

- **참여 구조 = 사용자 / 이용기관 2층**
  - **사용자**: 계좌 연결해 쓰는 사람 (개인 가능 — 이미 토스 등으로 사용 중)
  - **이용기관**: 송금·조회 API를 받아 **앱을 만드는 주체** → **순수 개인 불가**. 금융결제원과 이용계약을 맺은 **핀테크 사업자(법인/핀테크 분류업종 기업)·은행**만.
- **이용기관 요건**: 사업자 + 금융결제원 이용계약 + **보안점검·테스트** 통과. **이체(출금)** 는 조회보다 무거움(보증보험·정산·보안 요건 추가).
- **전자금융업 등록**이 필요한 모델이면: **법인 + 자본금 20억 + 재무건전성(부채비율 200% 이내)**. (모든 이용기관이 요하진 않으나 이체는 문턱이 높음)
- ⚠️ 오해 주의: *"2025년부터 개인→법인 확대"* 뉴스는 **'사용자' 범위** 확대지 **이용기관 자격 완화가 아님.**

## 비용 (참고 — 돈보다 자격이 관문)

- **이체**: 건당 ~40~50원, **영세 스타트업 ~20~30원** (기존 펌뱅킹 400~500원 대비 90%↓)
- **조회**(잔액·거래내역): 건당 10~30원대 → 추가 1/3 인하
- (2019~2020 도입 기준 수치, 현재 변동 가능 — 추진 시 재확인)

## 개인/소규모가 시도할 수 있는 경로

1. **테스트베드로 연동 프로토타입은 지금도 가능** — `developers.kftc.or.kr` 모의 API로 이용계약 전 개발·테스트(실계좌·실거래는 운영계약 후)
2. **소규모 법인/개인사업자 설립 → 이용기관 신청** (실추진 시)
3. **은행 핀테크 육성 프로그램** — 우리Developer, 디노랩 등 스타트업 MVP 협업·API 지원
4. **B2B 인프라 제휴** — 이미 이용기관인 핀테크 인프라사의 이체 API를 빌려 쓰는 우회(비법인 현실 경로)

## 언제 다시 볼까 (트리거)

- 토스 프로토타입으로 **수요·반응이 검증**되었을 때
- **법인 설립 / 사내(LG) 스폰서십 / 핀테크 제휴** 중 하나라도 확보될 때
- 그 시점에 위 경로 1→(2 또는 4) 순으로 진행
